# 01 — Los 5 Eventos de Scrum

## Objetivos

- Conocer los 5 eventos, sus timeboxes y sus propósitos
- Entender qué artefacto se inspecciona y adapta en cada evento
- Visualizar el flujo completo de un Sprint

---

## 1. El Sprint — El contenedor de todos los eventos

Un **Sprint** es un período fijo de 1 a 4 semanas donde el equipo crea
un Incremento de valor. Todos los demás eventos ocurren dentro del Sprint.

**Reglas clave del Sprint:**
- No se cancelan los Sprints salvo que el Sprint Goal pierda vigencia
- El Sprint Goal no cambia una vez iniciado el Sprint
- Solo el Product Owner puede cancelar un Sprint

---

## 2. Sprint Planning

| Aspecto | Detalle |
| ------- | ------- |
| Timebox | Máximo **8 horas** para Sprint de 4 semanas |
| Participantes | Todo el Scrum Team |
| Artefacto | Sprint Backlog |
| Resultado | Sprint Goal + Sprint Backlog |

**Las 3 preguntas del Sprint Planning:**
1. ¿Por qué es valioso este Sprint? → **Sprint Goal**
2. ¿Qué puede hacerse en este Sprint? → selección del Product Backlog
3. ¿Cómo se hará el trabajo seleccionado? → descomposición en tareas

---

## 3. Daily Scrum

| Aspecto | Detalle |
| ------- | ------- |
| Timebox | **15 minutos** |
| Participantes | Developers (el SM puede asistir, no lidera) |
| Artefacto | Sprint Backlog |
| Resultado | Plan para las próximas 24 horas |

> El Daily Scrum es para los **Developers**. Su propósito es inspeccionar
> el progreso hacia el Sprint Goal y adaptar el plan si es necesario.

---

## 4. Sprint Review

| Aspecto | Detalle |
| ------- | ------- |
| Timebox | Máximo **4 horas** para Sprint de 4 semanas |
| Participantes | Scrum Team + Stakeholders |
| Artefacto | Incremento + Product Backlog |
| Resultado | Product Backlog actualizado, feedback recogido |

---

## 5. Sprint Retrospective

| Aspecto | Detalle |
| ------- | ------- |
| Timebox | Máximo **3 horas** para Sprint de 4 semanas |
| Participantes | Scrum Team completo |
| Artefacto | Sprint Backlog (puede recibir items de mejora) |
| Resultado | Plan de mejora con al menos 1 acción concreta |

---

## Checklist de Comprensión

- ¿Puedo nombrar el propósito único de cada evento sin leer la Scrum Guide?
- ¿Sé qué artefacto se inspecciona en cada evento?
- ¿Entiendo por qué el Daily Scrum es de los Developers, no del Scrum Master?
- ¿Sé qué acción tomar si el Sprint Goal se vuelve irrelevante a mitad del Sprint?
