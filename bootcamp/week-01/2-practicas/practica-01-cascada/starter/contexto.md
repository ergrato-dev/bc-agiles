# Contexto — El Sistema de Gestión de Turnos del Ministerio

## Escenario

El Ministerio de Transporte de un país latinoamericano licitó en 2018 el
desarrollo de un nuevo sistema de gestión de turnos para sus 47 oficinas
regionales. El contrato fue adjudicado por USD 1.2 millones con un plazo
de 18 meses.

## Línea de tiempo

**Meses 1–3: Relevamiento de requisitos**
El equipo de consultoría pasó tres meses entrevistando funcionarios de
5 oficinas piloto. Se produjo un documento de requisitos de 340 páginas
aprobado y firmado por el Director General.

**Meses 4–7: Diseño**
El equipo técnico diseñó la arquitectura completa: base de datos,
módulos, interfaces y flujos. El cliente no participó en esta etapa
porque "eso era trabajo técnico".

**Meses 8–14: Desarrollo**
Se construyó el sistema completo en silencio. El cliente recibía
informes de avance en porcentajes: "60% completado", "80% completado".
Nadie del Ministerio vio una pantalla real.

**Mes 15: Primera demo**
El cliente vio el sistema por primera vez. Problemas descubiertos:
- El flujo de turnos no contemplaba las colas presenciales (que
  representaban el 70% del volumen real)
- Los reportes solo exportaban a formato Excel 2003
- El sistema requería Internet estable, pero el 60% de las oficinas
  regionales tenía conectividad intermitente
- La interfaz requería capacitación de 2 semanas por usuario

**Meses 16–18: Crisis**
Se intentaron parches de último momento. El plazo se extendió 6 meses.
El costo final fue de USD 2.1 millones. Solo 12 de las 47 oficinas
adoptaron el sistema. En 2020 fue dado de baja.
