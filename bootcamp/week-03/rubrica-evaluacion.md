# Rúbrica de Evaluación — Semana 03

**Panorama de Frameworks Ágiles**

---

## Tipos de evidencia

| Tipo | Peso | Descripción |
| ---- | ---- | ----------- |
| 🧠 Conocimiento | 30% | Identificar características y diferencias entre frameworks |
| 💪 Desempeño | 40% | Práctica de análisis de contexto y selección de framework |
| 📦 Producto | 30% | Documento de selección y justificación del framework para el dominio asignado |

**Mínimo aprobatorio por tipo**: 70 puntos

---

## 🧠 Evidencia 1: Conocimiento (30%)

| Criterio | Excelente (100) | Satisfactorio (70) | Insuficiente (<70) |
| -------- | --------------- | ------------------ | ------------------ |
| Diferencia Scrum vs Kanban | Explica con ejemplos al menos 3 diferencias estructurales | Menciona 2 diferencias | Confunde los frameworks |
| Cuándo usar XP | Identifica contextos técnicos donde XP es superior a Scrum | Menciona prácticas de XP | No distingue XP de Scrum |
| SAFe en contexto | Explica para qué tipo de organización aplica SAFe | Menciona que SAFe es para escalar | No sabe qué significa SAFe |

**Preguntas de autoevaluación:**
- [ ] ¿Puedo explicar la diferencia entre Scrum y Kanban en 2 oraciones?
- [ ] ¿Sé en qué contexto un equipo elegiría XP sobre Scrum?
- [ ] ¿Entiendo por qué SAFe no es para equipos pequeños?
- [ ] ¿Reconozco que ningún framework funciona para todos los contextos?

---

## 💪 Evidencia 2: Desempeño (40%)

| Criterio | Excelente (100) | Satisfactorio (70) | Insuficiente (<70) |
| -------- | --------------- | ------------------ | ------------------ |
| Análisis de contexto | Identifica 4+ factores que determinan la selección | Identifica 2–3 factores | Selecciona sin analizar el contexto |
| Justificación técnica | Argumenta con características específicas del framework | Menciona ventajas genéricas | Argumentación superficial |
| Identificación de antipatrones | Detecta cuándo el equipo aplica el framework incorrecto | Detecta un antipatrón obvio | No detecta problemas |

---

## 📦 Evidencia 3: Producto (30%)

El producto entregable es un **documento de selección de framework** para el
dominio asignado. Debe incluir:

| Criterio | Descripción |
| -------- | ----------- |
| Análisis del contexto | Características del equipo, producto y organización |
| Framework seleccionado | Nombre y versión (ej: Scrum, Kanban, Scrumban) |
| Justificación | Mínimo 3 razones específicas para el dominio asignado |
| Alternativas descartadas | Por qué NO se eligieron los otros frameworks |
| Riesgos anticipados | Qué desafíos puede traer el framework elegido |
