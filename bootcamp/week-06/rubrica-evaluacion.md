# Rúbrica de Evaluación — Semana 06

**Tema**: Artefactos de Scrum

| Criterio | Insuficiente (0–59) | Satisfactorio (60–79) | Destacado (80–100) |
| -------- | ------------------- | --------------------- | ------------------- |
| **Conocimiento 🧠 30%** | No distingue los 3 artefactos ni sus compromisos | Identifica artefactos pero confunde sus compromisos | Explica artefacto + compromiso + quién lo gestiona para los 3 |
| **Desempeño 💪 40%** | No puede priorizar un Backlog ni definir DoD/DoR | Prioriza el Backlog con criterio básico y define DoD parcial | Backlog con criterios MoSCoW, DoD + DoR con ≥5 criterios verificables |
| **Producto 📦 30%** | Product Backlog sin User Stories, sin prioridad | ≥5 User Stories con prioridad pero sin estimación ni DoD | ≥10 User Stories, MoSCoW, DoD de equipo, DoR con criterios claros |

## Puntaje Mínimo: 70 puntos por tipo de evidencia

---

## Cuestionario de Conocimiento (30%)

1. ¿Cuál es el compromiso asociado al Product Backlog?
2. ¿En qué se diferencia un Sprint Backlog de un Product Backlog?
3. ¿Por qué la Definition of Done NO es una checklist de QA individual?
4. ¿Qué artefacto se inspecciona en el Sprint Planning?

---

## Criterios del Proyecto (30%)

- Las User Stories siguen formato correcto (Como / Quiero / Para)
- La priorización usa un criterio explícito (MoSCoW, valor de negocio, etc.)
- La DoD tiene criterios verificables, no subjetivos ("El código funciona" ❌)
- La DoR indica cuándo un ítem está listo para entrar al Sprint Planning
