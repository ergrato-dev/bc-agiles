# Práctica 01 — Product Backlog Desordenado

**Semana 06 — Artefactos de Scrum**
**Tiempo estimado**: 1.5 horas

---

## Contexto

> **Escenario**: **CivicPortal**, una empresa de tecnología para gobiernos
> locales, acaba de adoptar Scrum. El PO, Hernán, recibió una lista de
> 12 requerimientos de múltiples fuentes (el alcalde, el equipo técnico
> y ciudadanos). Los mezcló en un documento de texto sin ningún orden.

---

## Paso 1: Convertir requerimientos en User Stories

Hernán tiene esta lista de "requerimientos" mezclados:

```
1. Que los ciudadanos puedan pagar sus multas online
2. Migrar la base de datos a PostgreSQL
3. El sistema debería estar en la nube
4. Reportes mensuales de trámites para el alcalde
5. Los ciudadanos deben poder agendar turnos para el registro civil
6. Debe tener autenticación de dos factores
7. La app debería ser más rápida
8. Los funcionarios necesitan ver el historial de un ciudadano
9. Botón de exportar a Excel
10. Que los viernes se cierre a las 14:00
11. Integración con el sistema de facturación existente
12. El sistema no debería caerse
```

Clasifica cada ítem en: **User Story potencial** / **Tarea técnica** /
**Restricción de negocio** / **Epic sin descomponer**.

**Abre `starter/plantilla.md`** y completa la Sección 1.

---

## Paso 2: Redactar 3 User Stories completas

Elige 3 ítems clasificados como "User Story potencial" y redáctalos en
formato correcto con criterios de aceptación (mínimo 2 por story).

**Abre `starter/plantilla.md`** y completa la Sección 2.

---

## Paso 3: Ordenar el Backlog

De los 12 ítems originales, Hernán necesita priorizar para el primer
Sprint. Ordena los 5 ítems de mayor prioridad y justifica el criterio.

**Abre `starter/plantilla.md`** y completa la Sección 3.
