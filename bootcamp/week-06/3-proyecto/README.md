# Proyecto — Semana 06: Product Backlog Inicial

## Descripción

Crearás el primer Product Backlog de tu producto asignado. Incluirá User
Stories ordenadas por prioridad, una Definition of Done del equipo y una
Definition of Ready para el Sprint Planning.

---

## Criterios de Evaluación

| Criterio | Peso |
|----------|------|
| ≥10 User Stories en formato correcto | 25% |
| Priorización explícita con criterio documentado | 25% |
| DoD con ≥5 criterios verificables | 25% |
| DoR con ≥4 criterios verificables | 25% |
