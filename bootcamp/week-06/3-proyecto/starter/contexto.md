# Contexto para el Proyecto de la Semana 06

## Instrucción para el aprendiz

Adapta este proyecto a tu dominio asignado. El ejemplo genérico utiliza
una plataforma de gestión de reservas para espacios comunitarios.
Tu producto debe ser el de tu dominio real asignado.

---

## Situación genérica de referencia

Un equipo Scrum está comenzando el desarrollo de un nuevo producto digital.
El Product Owner tiene una lista informal de funcionalidades deseadas pero
nunca formó un Product Backlog estructurado.

El equipo tiene:
- 4 developers
- 1 PO con conocimiento del dominio pero sin experiencia en Scrum
- 1 SM nuevo que quiere hacer las cosas bien desde el principio

El proyecto comienza en 2 semanas. El cliente quiere ver algo funcional
en el primer Sprint Review.

---

## Restricciones del Proyecto

- El equipo trabaja Sprints de 2 semanas
- Se deben identificar mínimo 3 tipos distintos de usuarios (roles)
- La DoD debe refleja el nivel técnico real de un equipo junior-mid
- La DoR debe garantizar que el Sprint Planning no dure más de 3 horas
