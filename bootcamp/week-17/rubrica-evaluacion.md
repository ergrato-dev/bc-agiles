# Rúbrica de Evaluación — Semana 17

## Criterios de Evaluación

| # | Criterio | Puntaje máximo |
| - | -------- | -------------- |
| 1 | **Conocimiento** 🧠 — Identifica correctamente los 3 frameworks y sus diferencias | 30 pts |
| 2 | **Desempeño** 💪 — Selecciona el framework adecuado con argumentación sólida | 40 pts |
| 3 | **Producto** 📦 — Diseña un ART inicial coherente con el dominio asignado | 30 pts |
| | **Total** | **100 pts** |

## Detalle por Criterio

### Conocimiento 🧠 (30 pts)

| Indicador | Pts |
| --------- | --- |
| Define correctamente SAFe, LeSS y Nexus | 10 |
| Identifica roles exclusivos de cada framework (RTE, Area PO, NIT) | 10 |
| Distingue cuándo NO se debería escalar | 10 |

### Desempeño 💪 (40 pts)

| Indicador | Pts |
| --------- | --- |
| Justifica la elección del framework con criterios concretos | 15 |
| Diagrama de estructuras del equipo es coherente | 15 |
| Detecta al menos 1 antipatrón del escalado en el caso de estudio | 10 |

### Producto 📦 (30 pts)

| Indicador | Pts |
| --------- | --- |
| El ART tiene entre 3 y 5 equipos con roles definidos | 10 |
| El Program Backlog tiene mínimo 3 Epics relevantes al dominio | 10 |
| El Sprint Goal de cada equipo refuerza el PI Objective | 10 |

## Criterios de Aprobación

- Mínimo **70 puntos** en total
- No se aprueba con 0 en ningún criterio
- Entregas con dominios copiados de otros aprendices = 0 en Producto
