# 01 — Principios y Prácticas de Kanban

## Objetivos

- Comprender los 3 principios fundacionales de Kanban
- Conocer las 6 prácticas básicas del método Kanban
- Distinguir Kanban del tablero Kanban (el tablero es solo una herramienta)

---

## 1. Kanban no es un tablero

Un tablero Kanban es una herramienta de visualización. El **método Kanban**
es un sistema completo de gestión de flujo con principios, prácticas y
métricas para optimizar la entrega continua de valor.

**Origen**: El concepto viene del sistema de producción de Toyota (TPS),
donde "kanban" significa "tarjeta visual". David Anderson lo formalizó
para el desarrollo de software en 2010.

---

## 2. Los 3 Principios Kanban

### Principios de Cambio
1. **Empezar con lo que tienes**: no hay que reorganizar el equipo antes
   de adoptar Kanban. Se visualiza el flujo actual primero.
2. **Acordar buscar la mejora incremental**: no hay grandes transformaciones;
   el cambio es continuo y pequeño.
3. **Respetar los procesos y roles actuales**: Kanban no destruye lo existente,
   lo hace visible para mejorarlo.

---

## 3. Las 6 Prácticas de Kanban

| Práctica | Qué hace |
| -------- | -------- |
| **1. Visualizar** | Hace visible el flujo de trabajo en el tablero |
| **2. Limitar el WIP** | Establece un máximo de ítems en progreso por columna |
| **3. Gestionar el flujo** | Mide y optimiza cómo se mueve el trabajo |
| **4. Políticas explícitas** | Define reglas claras de qué significa "en progreso" o "hecho" |
| **5. Loops de feedback** | Reuniones de cadencia para revisar y adaptar |
| **6. Mejorar colaborativamente** | El equipo mejora el sistema de forma incremental |

---

## 4. Estructura básica del tablero

El tablero más simple tiene 3 columnas:
**To Do → In Progress → Done**

Un tablero más maduro refleja el flujo real del equipo:
**Backlog → Análisis → Desarrollo → Revisión → Staging → Done**

Cada columna puede tener una **sub-columna "En proceso" / "Listo"** para
distinguir el trabajo activo del que espera ser jalado por la siguiente etapa.

---

## Checklist de Comprensión

- ¿Entiendo que Kanban parte del flujo actual, sin imponer roles nuevos?
- ¿Puedo nombrar las 6 prácticas sin leer el material?
- ¿Sé por qué limitar el WIP es más importante que trabajar más rápido?
- ¿Comprendo la diferencia entre el tablero Kanban y el método Kanban?
