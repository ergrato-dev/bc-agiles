# Práctica 01 — Scrum Master vs Project Manager

**Semana 04 · Roles de Scrum**
**Tiempo estimado**: 75 minutos

---

## Objetivo

Distinguir qué comportamientos pertenecen al Scrum Master y cuáles al Project
Manager clásico, a partir de situaciones concretas.

---

## Escenario base

**PaperMap** es una empresa de cartografía digital con equipo de 8 personas.
Hace 3 meses adoptaron Scrum. **Marcelo** fue Project Manager durante 6 años
y ahora es Scrum Master. Lee lo que hace Marcelo cada semana.

---

### Paso 1: Acciones de Marcelo (semana típica)

1. Cada lunes envía un reporte al CTO con el avance de las tareas del Sprint.
2. En el Sprint Planning desglosa las User Stories en tareas técnicas y las asigna.
3. Cuando el equipo tiene un bloqueo técnico, lo resuelve él mismo.
4. Habla directamente con el cliente para explicar por qué no se entregó algo.
5. En el Daily, pregunta a cada persona: "¿Cuánto llevas hecho de tu tarea?".
6. Organiza la Sprint Review preparando la demo y coordinando los asistentes.
7. Entrena al nuevo dev junior explicando las prácticas de código.
8. Detecta que el PO no tiene tiempo para el backlog y plantea el problema al Director.

**Abre `starter/plantilla.md`** y clasifica cada acción en `PASO 1`.

---

### Paso 2: Conversación con el CTO

El CTO llama a Marcelo: "Necesito que el equipo pause la funcionalidad X y
trabaje en el bug urgente Y esta semana. Es crítico para el cliente enterprise."

**Completa el `PASO 2` en la plantilla.**

---

### Paso 3: El dilema del nuevo dev

El dev junior lleva 3 días bloqueado con una task. No pidió ayuda al equipo.
Marcelo se da cuenta en el Daily. El dev dice: "No quería interrumpir a los seniors."

**Completa el `PASO 3` en la plantilla.**
