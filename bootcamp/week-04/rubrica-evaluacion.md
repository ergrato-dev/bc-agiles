# Rúbrica de Evaluación — Semana 04

**Roles de Scrum**

---

## Tipos de evidencia

| Tipo | Peso | Descripción |
| ---- | ---- | ----------- |
| 🧠 Conocimiento | 30% | Identificar responsabilidades y antipatrones de cada rol |
| 💪 Desempeño | 40% | Prácticas de análisis de situaciones reales de roles |
| 📦 Producto | 30% | Documento de roles adaptado al dominio asignado |

**Mínimo aprobatorio por tipo**: 70 puntos

---

## 🧠 Evidencia 1: Conocimiento (30%)

| Criterio | Excelente (100) | Satisfactorio (70) | Insuficiente (<70) |
| -------- | --------------- | ------------------ | ------------------ |
| Diferencia SM vs PM | Describe 3+ diferencias con ejemplos concretos | Menciona 2 diferencias | Confunde los roles |
| Responsabilidades del PO | Explica qué decide el PO y qué no puede delegar | Menciona el backlog | No diferencia PO de Analista de Negocio |
| Cross-funcionalidad | Explica por qué el equipo debe tener todas las habilidades | Lo menciona sin ejemplos | No comprende el concepto |

**Preguntas de autoevaluación:**
- [ ] ¿Puedo explicar por qué el SM no puede asignar tareas?
- [ ] ¿Sé quién puede cancelar un Sprint y en qué circunstancias?
- [ ] ¿Entiendo por qué "cross-functional" no significa que todos hacen todo?
- [ ] ¿Reconocería el antipatrón "PO teléfono" en un equipo real?

---

## 💪 Evidencia 2: Desempeño (40%)

| Criterio | Excelente (100) | Satisfactorio (70) | Insuficiente (<70) |
| -------- | --------------- | ------------------ | ------------------ |
| Clasificación de comportamientos | Clasifica correctamente 7/8 acciones | Clasifica 5–6 correctamente | < 5 correctas |
| Manejo de presión del PO | Propone respuesta que protege el Sprint Goal sin conflicto | Responde correctamente pero de forma genérica | Cede ante la presión sin argumento |
| Plan de saneamiento del backlog | 3 acciones ordenadas y concretas | 2 acciones con mínimo detalle | No propone acciones concretas |

---

## 📦 Evidencia 3: Producto (30%)

| Criterio | Descripción |
| -------- | ----------- |
| Roles adaptados al dominio | Las responsabilidades son específicas, no copiadas de la Scrum Guide |
| Antipatrones anticipados | 1 antipatrón por rol, con contexto del dominio |
| Mapa de interacciones | Describe fricciones reales esperadas en el dominio |
| Originalidad | Sin copias de otros aprendices; uso del dominio propio |
